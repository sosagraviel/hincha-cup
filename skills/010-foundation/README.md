# 010 - Foundation Skills

**Priority**: ⭐⭐⭐⭐⭐ HIGHEST
**Usage**: Project bootstrap, onboarding, architectural understanding

## Purpose

Foundation skills establish project fundamentals and generate base configuration. These are typically used once per project or when joining a new codebase.

## Skills in This Group

### initialize-project
**Purpose**: Deep codebase analysis and Claude Code configuration generator

**What it does**:
- Spawns 4 parallel analyzer agents (haiku) to analyze structure, auth, DevOps, conventions
- Spawns 1 architect agent (opus) to synthesize findings
- Generates `.claude/CLAUDE.md` (commands, stack, conventions)
- Generates `.claude/skills/project-context/SKILL.md` (hard-to-discover flows)

**When to use**:
- Setting up Claude Code for a new/legacy codebase
- Refreshing project documentation after major refactors
- Onboarding new team members

**Agents**: Uses 5 agents from `.claude/agents/initialize-project-agents/`

---

### project-context
**Purpose**: Hard-to-discover architectural knowledge

**What it contains**:
- Request lifecycle (middleware → guards → interceptors → controller)
- Auth flow (token → validation → caching → guards)
- Real-time architecture (mutation → queue → Socket.IO)
- Error handling chain (exceptions → filters → responses)
- Non-obvious patterns (guard stacking, naming strategies)

**When to use**:
- Implementing features touching auth, real-time, or cross-cutting concerns
- Understanding complex multi-file flows
- Debugging non-obvious behavior

**Generated by**: `initialize-project` skill

---

### analyze-requirements
**Purpose**: Jira ticket context fetching and implementation planning

**What it does**:
- Fetches ticket from Jira with external documentation
- Analyzes requirements and dependencies
- Creates detailed implementation plan with:
  - File changes needed
  - Risks and edge cases
  - Implementation steps
  - Testing requirements

**When to use**:
- Before starting feature implementation
- Understanding ticket scope and complexity
- Creating work estimates

## Workflow

### Initial Project Setup
```
1. Run initialize-project
   → Generates CLAUDE.md + project-context skill

2. Review generated files
   → Verify accuracy, add missing details

3. Load project-context skill
   → /project-context to activate deep knowledge
```

### Starting New Feature
```
1. Run analyze-requirements
   → Get ticket context + plan

2. Review plan with team
   → Validate approach

3. Proceed to 020-development-workflow
   → Implement the feature
```

## Best Practices

1. **Run initialize-project first** - Always start new projects here
2. **Keep CLAUDE.md updated** - Refresh after major architectural changes
3. **Use project-context liberally** - Load it before complex features
4. **Verify plans** - Review analyze-requirements output before coding
5. **Don't skip foundation** - Proper setup prevents confusion later
