# Release 28/04/2026

## Release Notes

### New

- **Codex support.** You can now run the framework with OpenAI Codex CLI as an alternative to Claude. Pick one with `--provider claude` or `--provider codex` on `initialize-project`.
- **New skills auto-installed.** Scala support, GitHub CLI mastery, Playwright E2E automation, and `fetch-ticket-context` is now always available.
- **Better project detection.** Phase 1 analyzers now detect automation (Make, npm scripts, etc.) and produce a richer stack profile.

### Changed

- **Slash commands are gone.** `/create-sdd-ticket`, `/implement-ticket`, `/start-task`, `/end-task`, `/list-tasks`, and `/switch-task` no longer exist as slash commands. They are now unified **skills** — invoke them just like you did before.
- **Sync cleans up after itself.** Running `sync-framework-resources.sh` now deletes the legacy `.claude/commands/` directory automatically.
- **Stricter file-counter and path exclusion** in Phase 1 (no more false positives from `node_modules`, build outputs, etc.).
- **Improved CLAUDE.md generation** (unified Phase 3 prompts, no duplicated sections).

### Fixed

- Default-provider logic when neither flag nor env var is set.
- Several `initialize-project` cases (Now supporting API KEYs authentication).
- Duplicate CLAUDE.md warning before init clobbers an existing one.

### Internal (no user action required)

- Always-on per-attempt debug/transcript capture under `.<provider>-temp/.../debug/`.
- ESLint + Prettier + Husky now enforced in `orchestration/`.
- GitHub Actions CI for lint/build/unit tests.

## What should I do? — pick your scenario

### Scenario 1 — "I'm using already using QAF with Claude provider, everything works, I just want the latest version"

**Do this:**

```bash
cd /path/to/your/project/qubika-agentic-framework
git pull
cd ..
./qubika-agentic-framework/scripts/sync-framework-resources.sh
```

That's it. You'll get all the new skills, updated agents, and the obsolete `.claude/commands/` folder will be cleaned up automatically. Your `CLAUDE.md` and project analysis are left alone — they still work.

### Scenario 2 — "I want to try Codex (OpenAI) instead of Claude"

You **must re-initialize** for this — sync alone cannot switch providers because the config files live in different directories (`.claude/` vs `.codex/`).

```bash
./qubika-agentic-framework/scripts/initialize-project.sh --provider codex
```

Your existing `.claude/` directory will stay untouched; you can keep both side-by-side or delete `.claude/` afterwards.

### Scenario 3 — "I want my project's `CLAUDE.md` and analysis to use the new improvements"

This is **optional**. The new Phase 1 detection (automation, file-counter fixes) and Phase 3 CLAUDE.md prompts only apply on a fresh `initialize-project` run. If you want them:

```bash
# back up your current .claude folder first if you've edited it
cp -r .claude .claude.bak

./qubika-agentic-framework/scripts/initialize-project.sh --provider claude
```

If you're happy with your current `CLAUDE.md`, **skip this** — Scenario 1 is enough.

### Scenario 4 — "I customized a skill or agent file and don't want my changes overwritten"

Sync already protects you. Any skill/agent marked `managed_by_framework: false` (which happens automatically when you edit one and the file hash diverges) is **skipped** during sync. Just run Scenario 1; your edits survive.

### Scenario 5 — "I'm not sure if I customized anything"

Run Scenario 1. Sync creates a timestamped backup at `.claude-backups/<timestamp>/` before doing anything, so you can always roll back.

### Scenario 6 - "I want to use API KEY authentication instead of CLI auth"
You can now authenticate using environment variables instead of CLI commands.

#### Scenario 6.1 — "Using Claude"
```bash
# logout of CLI auth if you were using it
claude

# inside the CLI, run the /logout command to clear CLI auth
/logout

# set your API key as an environment variable
export ANTHROPIC_API_KEY=sk-ant-your-key

# Now run Scenario 1 or 3 to sync or re-init with API key auth
./qubika-agentic-framework/scripts/initialize-project.sh --provider claude
```

#### Scenario 6.2 — "Using Codex"
```bash
# logout of CLI auth (if you were using it)
codex

# inside the CLI, run the logout command to clear CLI auth
/logout

# set your API key as an environment variable
export OPENAI_API_KEY=sk-...

# Now run Scenario 2 or 3 to init with Codex and API key auth
./qubika-agentic-framework/scripts/initialize-project.sh --provider codex

## Decision flow (one-liner)

| Your situation                            | What to run                                  |
| ----------------------------------------- | -------------------------------------------- |
| Default — just want the update            | `sync-framework-resources.sh`                |
| Want to switch to Codex                   | `initialize-project.sh --provider codex`     |
| Want the improved analysis/CLAUDE.md      | `initialize-project.sh` (re-init)            |
